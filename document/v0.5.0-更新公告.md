# Kunkun v0.5.0 更新公告

> 发布日期: 2026-07-07
> 版本: v0.5.0
> 状态: ThinkBlock 过程评测 — 基于 AgentThink 论文的过度思考检测

---

## 一、版本概述

v0.5.0 是 Kun 的第一个 DS 专属创新版本。利用 DSv4 暴露 `reasoning_content` 的独有特性，实现了**ThinkBlock 过程评测**——每次任务执行完，自动用 flash 模型对思考过程进行三维度量打分。

这是 Claude Code 和 Hermes 做不到的能力——它们看不到模型的推理过程，只有 DSv4 的 ThinkBlock 让"思考质量评测"成为可能。

---

## 二、核心功能：ThinkBlock 三维度评测

借鉴 [AgentThink (Cuadron et al., 2025)](https://sky.cs.berkeley.edu/project/agentthink/) 论文的评分 prompt，每次 Agent 执行结束后自动分析：

| 维度 | 检测什么 | 评分 |
|------|---------|------|
| **分析瘫痪** | 想得多做得少，重规划轻行动 | 0-10 (低好) |
| **流氓操作** | 工具出错后不等反馈连续调多个工具 | 0-10 (低好) |
| **过早放弃** | 第一次失败就认输，或没验证就声明成功 | 0-10 (低好) |

### 使用方式

verbose 模式下自动显示（`KUN_VERBOSE=1`）：

```
📊 ThinkBlock 过程评测
   综合: 2/10 🟢
   分析瘫痪: 1/10 🟢
   流氓操作: 3/10 🟢
   过早放弃: 4/10 🟡
   诊断: 模型在遇到错误后有轻微放弃倾向，但整体互动良好
   分析事件: 47 个 | 评测模型: deepseek-v4-flash
```

### 技术细节

- 使用 flash 模型（低成本）进行评估
- 每次执行后异步运行，不阻塞主流程
- 评分报告输出到 stderr，不进入 Agent 对话循环
- 论文原文的评分 prompt 拆为三维独立打分
