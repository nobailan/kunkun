# v0.9 Workflow + Cron 计划

> DSv4 适配 + 最小 Token 浪费原则

---

## 一、核心原则

### 1.1 Token 节省策略

| 策略 | 实现 |
|------|------|
| **Cron 任务复用 Frozen Snapshot** | 定时任务的 System Prompt 只构建一次，后续复用 |
| **Workflow pipeline 默认（非 barrier）** | Item A 进 stage 2 时 Item B 还在 stage 1，不等最慢的 |
| **轻任务走 flash** | Cron 健康检查/简单查询自动路由到 flash |
| **上下文不膨胀** | Cron 任务独立 session，不带历史对话 |
| **共享 Memory/Skill** | 子 Agent 读父 Agent 的 Frozen Snapshot，不重复加载 |
| **结果增量合并** | 不拼接全文，用 summary 替代 |

### 1.2 DSv4 适配

- **ThinkBlock 优化**：Workflow 拆解阶段用 ThinkBlock，执行阶段跳过 ThinkBlock
- **成本路由自动生效**：Cron 任务默认走 flash，只在检测到复杂关键词时切 pro
- **MLA 宽松压缩**：Cron 长任务保留更多上下文（DSv4 便宜）

## 二、实现内容

### 2.1 Cron 调度器（Phase 1）

```
src/kunkun/cron/
├── __init__.py
├── scheduler.py       # CronScheduler (asyncio)
└── parser.py          # Cron 表达式解析
```

**API**：

```python
from kunkun.cron import CronScheduler

scheduler = CronScheduler()

@scheduler.task("0 9 * * 1-5")
async def daily_standup():
    result = await run_agent("汇总昨日变更")
    return result

scheduler.start()  # 后台运行
```

**功能清单**：
- 标准 5 字段 cron 解析（分 时 日 月 周）
- 任务持久化（`.kun/cron/tasks.json`）
- 失败重试（指数退避，3 次）
- 并发控制（Semaphore，默认 1）
- 状态追踪（上次执行/成功/失败/耗时）
- 跳过策略（上次未完成 → 跳过本次）
- `--verbose` 模式显示下次触发时间

**Token 优化**：
- Cron 任务 Agent 复用 Frozen Snapshot
- 默认走 flash 模型

### 2.2 Workflow 引擎（Phase 2）

```
src/kunkun/workflow/
├── __init__.py
├── engine.py          # @workflow, agent(), parallel(), pipeline(), phase()
└── registry.py        # Workflow 注册
```

**API**：

```python
from kunkun.workflow import workflow, agent, parallel, pipeline, phase

@workflow(name="code-review-nightly")
async def nightly_review():
    phase("扫描")
    files = await agent("grep 最近修改的 Python 文件")
    
    phase("审查")
    results = await pipeline(
        files,                                    # 待审查文件列表
        lambda f: agent(f"审查 {f} 的安全问题"),   # stage 1
        lambda r: agent(f"验证 {r}"),             # stage 2
    )
    
    phase("汇总")
    return await agent(f"汇总: {results}")

# 注册到 cron
scheduler.register_workflow("0 2 * * *", nightly_review)
```

**原语设计**：

| 原语 | 行为 | Token 优化 |
|------|------|-----------|
| `agent(prompt, role)` | 启动角色化子 Agent | 共享 Frozen Snapshot |
| `parallel(tasks)` | **屏障** — 等全部完成 | 各自独立 session |
| `pipeline(items, ...stages)` | **非屏障默认** — 流水线 | Item A stage 2 时 Item B stage 1 |
| `phase(title)` | 进度标记 | 仪表盘可视化 |

### 2.3 仪表盘增强（Phase 3）

- Cron 任务状态卡片（下次触发时间/上次结果/成功率）
- Workflow 执行时间线
- Cron 任务历史趋势图

## 四、不做的

- 云端 Routine（需要外部依赖，v1.0 后考虑）
- Cron 秒级精度（分钟级够用）
- 图形化 Workflow 编辑器（FlowForge 的职责）
- 分布式 Cron 锁（单机够用）

## 五、测试标准

- Cron 表达式解析正确性（5/10/15/30 分钟间隔各测一轮）
- 任务跳过策略（上一个未完成 → 跳过）
- Workflow pipeline 非屏障验证
- 91/91 回归无异常
