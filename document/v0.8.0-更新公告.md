## 一、版本概述

v0.8.0 实现了评测仪表盘——将 v0.5-v0.7 积累的评测数据（ThinkBlock 过程分析 + AdaRubric 任务评分）以可视化方式呈现。

现在你可以**看到** Agent 的思考质量和任务完成度，而不只是"感觉"它做得好不好。

---

## 二、评测仪表盘

### 入口

```bash
kunkun --dashboard
```

自动生成 HTML 并在浏览器打开。每次运行覆盖旧文件，数据实时更新。

### 三大板块

| 板块 | 数据来源 | 呈现方式 |
|------|---------|---------|
| **ThinkBlock 过度思考评分趋势** | `evaluations.jsonl` | 柱状图（绿/黄/红），每个柱子 = 一次会话的综合评分 |
| **AdaRubric 任务维度评分** | `evaluations.jsonl` | 条形图（前 15 维度平均分） |
| **会话列表 + 工具分布** | `.kun/reports/*.json` | 表格（最近 20 会话）+ 条形图（Top 10 工具） |

### 技术特点

- **零外部依赖** — 纯 Python 生成 HTML，不依赖 matplotlib/echarts
- **单文件** — 一个 HTML 包含所有 CSS/JS
- **深色主题** — GitHub 风格暗色配色
- **缓存防护** — 每次生成唯一文件名，浏览器不会显示旧数据

## 三、评测数据持久化

### 产出

```
.kun/
├── reports/{session_id}.json     ← 事件时间线 (v0.2)
├── dashboard-{timestamp}.html    ← 仪表盘 (v0.8)
├── evaluations.jsonl             ← 评测记录 (v0.8)
└── logs/kunkun.log               ← 运行日志 (v0.7)
```

### evaluations.jsonl 格式

每行一条 JSON，包含：

```json
{
  "session_id": "...",
  "timestamp": "2026-07-10T10:30:00",
  "thinking_eval": {
    "analysis_paralysis": 1,
    "rogue_actions": 0,
    "premature_disengagement": 0,
    "overall": 1,
    "summary": "模型高效执行，无过度思考"
  },
  "task_eval": {
    "dimensions": [
      {"name": "搜索完整性", "score": 2, "max": 3},
      {"name": "结果准确性", "score": 3, "max": 3}
    ],
    "overall": 2.5,
    "summary": "搜索准确但覆盖不全"
  }
}
```

## 四、FlowForge 接入

### FlowForgeAdapter

```python
from kunkun.core.flowforge_adapter import FlowForgeAdapter, FlowForgeTask

adapter = FlowForgeAdapter()
result = await adapter.execute(FlowForgeTask(
    task_id="eval-001",
    prompt="分析 src/kunkun/core/ 的架构",
))
print(result.to_dict())
# → {"task_id": "...", "success": true, "turns": 2,
#    "tokens": {"input": 15207, ...}, "cost_usd": 0.0097, ...}
```

### CLI 模式

```bash
kunkun --flowforge "分析 src/kunkun/core/ 的架构"
# → JSON 输出, FlowForge 可直接解析
```

### 异步信箱系统

```
子 Agent (worker 线程)                 父 Agent (async 循环)
  ├── text chunk → Mailbox.put()  →  await Mailbox.get() → yield
  ├── result     → Mailbox.put()  →  保存到 _last_result
  └── done       → Mailbox.put()  →  break

AgentBus (pub/sub, 多订阅者)
  ├── Agent A: bus.subscribe("A", callback, filter_fn)
  ├── Agent B: bus.subscribe("B", callback)
  └── Agent C: bus.publish(msg) → 所有匹配订阅者收到消息
```

| | Claude Code | Kunkun v0.8 |
|---|:---:|:---:|
| 子→父通信 | async mailbox | AgentMailbox |
| Agent 间通信 | pub/sub | AgentBus (多订阅者 + 过滤器) |
| 结构化结果 | ✅ | ✅ (tokens, turns, cost) |

### 批量执行

```python
results = await adapter.execute_batch([
    FlowForgeTask(task_id="1", prompt="..."),
    FlowForgeTask(task_id="2", prompt="..."),
], max_concurrency=3)
```

## 五、关键修复

| 问题 | 修复 |
|------|------|
| 仪表盘会话按 UUID 排序（乱序） | 改为按 `started_at` 时间降序 |
| AdaRubric 评测永远失败（空轨迹） | `_build_task_trajectory()` 移到 `reset()` 之前 |
| AdaRubric 走代理导致 API 不通 | task_eval 改为直连 DeepSeek API |
| ThinkBlock 评分 JSON 被截断 | max_tokens 512 → 1024 |
| AdaRubric 评分 JSON 被截断 | max_tokens 1024 → 2048 |
| 评测数据没持久化 | `_run_thinking_eval` 追加写入 `evaluations.jsonl` |
| `exit` 后评测被取消 | `close()` 改为 `await` 而非 `cancel` |
| 浏览器缓存旧仪表盘 | 唯一文件名 + no-cache meta 标签 |
| System Prompt 不显示当前日期 | 运行环境加入 `当前日期` 字段 |
| Tavily 走代理导致搜索失败 | Tavily API 调用改为直连 |

## 七、Agent 编排三方对比

详见 [AgentTeam 三方对比](./AgentTeam三方对比.md)。

核心差异：
- **角色化隔离** — Explorer/Coder/Reviewer/Planner 各有工具白名单
- **双通道通信** — AgentMailbox (1:1) + AgentBus (pub/sub)
- **评测内置** — ThinkBlock + AdaRubric 自动评分
- **显式任务拆解** — Planner 生成 JSON 计划 → 按批次委派
