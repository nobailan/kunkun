## 一、版本概述

v0.7.0 有三个交付，两个来自论文：

1. **Prompt 粒度编译器** — DS 专属创新。同一任务根据目标模型（Pro/Flash/R1）自动适配 System Prompt 粒度。
2. **AdaRubric 任务评测** — 借鉴 AdaRubric 论文。执行后动态生成任务专属评分维度并打分。
3. **事件分发架构重构** — agent_loop.py if-elif 替换为 `EventDispatchChain` + `StopReasonRouter`。

三篇论文的 prompt 全部落地：AgentThink（过程质量）+ AdaRubric（结果质量）+ GRPO（择优）。

## 二、Prompt 粒度编译器

### 设计原理

不同 DS 模型有不同的指令遵循特性，不能用同一套 prompt：

| 模型 | 特性 | Prompt 策略 |
|------|------|------------|
| **Pro** | 推理强，给方向就行 | 简洁版，"理解意图后直接执行" |
| **Flash** | 轻量，需要明确边界 | 详尽版，列 7 条操作规则 + 3 条禁止事项 |
| **R1** | 擅长拆解问题 | 步骤化版，理解→规划→执行→验证 四步法 |

### 自动适配

- 启动时根据 `KUN_MODEL` 自动选择 prompt 粒度
- 成本路由切换模型时自动重新编译 prompt
- 子 Agent 任务 prompt 也按模型适配
- Flash 模型额外注入工具使用提示（edit 需要多少上下文、grep 范围控制等）

## 三、事件分发架构

### EventDispatchChain

```
旧: if-elif-elif-elif (10 分支)
新: chain.on(TYPE, handler).on(TYPE, handler)...

每个 handler:
  - 接收 (event, ctx)
  - 更新共享 EventContext (all_text, tool_uses, stop_reason...)
  - 返回 True (已处理) / False (传给下一个)
```

### StopReasonRouter

```
旧: if stop_reason == "end_turn": ... elif "tool_use": ...
新: router.register("end_turn", lambda) → router.route(ctx)

每个策略:
  - 返回 StopReasonResult(action, message, final_result)
  - action: "break" | "continue" | "break_warn"
```

### 具体效果

- `run()` 圈复杂度从 ~20 降到 ~8
- 新增事件类型无需修改 agent_loop.py——注册 handler 即可
- `event_dispatch.py` 独立可测

## 四、AdaRubric 任务评测

### 设计原理

借鉴 AdaRubric (Ding et al., 2025)：不为所有任务用同一套评分标准。搜索机票的 Agent 和修 bug 的 Agent 不能用同一把尺子。

```
执行后 → flash 模型分析任务内容 → 动态生成评分维度 → 逐维度 0-3 打分

  任务"修 issue #42":  bug已修复 2/3, 向后兼容 3/3, 测试覆盖 0/3
  任务"搜索文档":     找到主文档 3/3, 版本号正确 1/3, 中文结果 3/3
```

### 和 ThinkBlock 评测的关系

| | ThinkBlock 评测 | AdaRubric 评测 |
|---|:---:|:---:|
| 评什么 | **过程**（怎么想的） | **结果**（做得怎么样） |
| 评分方式 | 固定三维度 | 任务动态生成维度 |
| 论文来源 | AgentThink | AdaRubric |

两者合并输出，verbose 模式下显示完整 Agent 评估。
