## 三、核心模块说明

### 3.1 错误恢复

```
ErrorClassifier → classify(HTTPStatusError/TimeoutError/...) → retryable | fatal
RetryPolicy     → base=1s, max=60s, retries=3, jitter=0.3
async_retry()   → 自动重试 429/5xx/timeout，4xx 直接报错
```

### 3.2 权限管道

三层门禁：
1. **Deny list** — 15 条危险命令模式（rm -rf /, sudo, curl|bash, fork bomb...）
2. **Workspace 边界** — 读/写操作必须在 workspace 目录内
3. **Permission mode** — default/accept_edits/bypass

### 3.3 执行日志

```
ExecutionLogger → 收集 Event → flush() → .kun/reports/{session_id}.json
```

每次会话自动生成完整事件时间线报告。

### 3.4 记忆系统

```
.kun/memory/*.md → YAML frontmatter + Markdown body
MemoryManager → load() / select(prompt) / save() / search()
```

启动时自动加载，按关键词匹配选择 ≤5 条注入 System Prompt。

### 3.5 成本路由

```
prompt → classify_task() → ModelTier.LIGHT | HEAVY
       → BudgetTracker → can_use_pro()?
       → 最终模型: flash | pro
```

三层漏斗：规则匹配（v0.2 保留）→ Flash（简单任务）→ Pro（复杂任务）

## 四、遇到的问题与解决方案

### 问题 1: Python 3.9 `X | None` 语法不支持

**现象**: `TypeError: unsupported operand type(s) for |: 'type' and 'NoneType'`

**解决**: 在 registry.py 中使用 `from __future__ import annotations` + `Optional[T]`

### 问题 2: asyncio.Event() 在 Python 3.9 需要事件循环

**现象**: `RuntimeError: There is no current event loop in thread 'MainThread'`

**解决**: 改为 lazy init，在 `run()` 方法中按需创建 `asyncio.Event()`

### 问题 3: YAML frontmatter 解析 `metadata:` 空值

**现象**: `metadata:` 行被解析为 key="metadata", value=""，后续 `setdefault` 返回字符串而非 dict

**解决**: 用 `in_metadata` 标志位跟踪是否在 metadata 块内，准确区分父子键

### 问题 4: 异步测试在 pytest + anyio 下自动双参数化

**现象**: 使用 `@pytest.mark.anyio` 后，测试自动用 `[asyncio]` 和 `[trio]` 双后端执行，trio 未安装导致失败

**解决**: 运行测试时添加 `-k "not trio"` 排除 trio 后端
