## 一、版本概述

v0.3.2 是 v0.3 的架构升级。核心变化：**从"静态 Skill 模板 + 手动 Memory"进化为完整的自改进学习系统**。

今天上午的核心收获：深入理解了 Hermes 的 Memory 和 Skill 不是两个独立功能，而是一个统一系统的两面——"学到了什么事实"→ Memory，"学到了什么模式"→ Skill。两者共享相同的文件格式、相同的 System Prompt 注入机制、相同的自改进闭环。

v0.3.2 把这个闭环完整实现了。

## 三、架构变更

### 3.1 Frozen Snapshot — 保护 prefix cache

```
会话启动 → 首次 run() 时:
  ├── 加载全部 Memory 全文 → 注入 System Prompt
  ├── 加载全部 Skill 全文 → 注入 System Prompt
  └── 冻结 → 整个会话不再变

Turn 1: cache write → Turn 2+: cache HIT ✅
Mid-session 写入 → 只落盘 → 下次会话生效
```

### 3.2 记忆与 Skill 的统一模型

| | Memory | Skill |
|---|--------|-------|
| 本质 | "是什么"（事实） | "怎么做"（约定） |
| 写入工具 | `remember(target="memory")` | `remember(target="skill")` |
| 注入方式 | Frozen Snapshot 全文 | Frozen Snapshot 全文 |
| 检索 | `recall` 工具 (n-gram + fallback) | 全文在 System Prompt，Agent 自动遵守 |
| 进化 | background_review 自动提取 | background_review 自动修补/创建 |
| 生命周期 | 手动管理 | Curator (active→stale→archived) |

### 3.3 批量记忆/Skill 写入

`remember` 工具新增 `operations` 数组：

```json
{
  "operations": [
    {"action": "remove", "target": "memory", "old_name": "old-pref"},
    {"action": "add", "target": "skill", "name": "coding-rules", "content": "...", "triggers": ["Python", "编码"]}
  ]
}
```

一次性完成增删改，减少 LLM 往返。

### 3.4 Skill 进化质量保障

反思引擎（background_review）获取**匹配 Skill 的全文**进行判断，确保能精确发现"这个 Skill 的第 N 条规则跟用户这次说的冲突，需要修补"。

主对话省 token（Frozen Snapshot 只付一次），反思给足信息（全文对比），两端各取最优。

## 八、架构全景

```
                       ┌─────────────────────┐
                       │  Frozen Snapshot     │
                       │  首次 run() 构建      │
                       │  Memory + Skill 全文  │
                       │  后续轮次复用 (cache)  │
                       └──────────┬──────────┘
                                  │ 注入
                                  ▼
  ┌─────────┐            ┌──────────────┐      ┌──────────┐
  │  User   │───────────▶│  Agent 执行   │─────▶│  Result  │
  └─────────┘            └──────┬───────┘      └──────────┘
                                │
                                │ stop_reason="end_turn"
                                ▼
                       ┌────────────────────────┐
                       │   background_review     │
                       │   (flash, async)        │
                       │   → memory auto-save    │
                       │   → skill auto-patch    │
                       │   (获取 Skill 全文对比)  │
                       └────────┬───────────────┘
                                │
                    ┌───────────┴────────────┐
                    ▼                        ▼
            ┌──────────────┐        ┌──────────────┐
            │ Memory        │        │ Curator       │
            │ .kun/memory/  │        │ active→stale   │
            │ 下次会话生效   │        │ →archived      │
            └──────────────┘        └──────────────┘
```
