# Kunkun v0.6.0 更新公告

> 发布日期: 2026-07-07
> 版本: v0.6.0
> 状态: GRPO 多版本生成 — 3 路径并行 + LLM-as-Judge 择优

---

## 一、版本概述

v0.6.0 实现了 GRPO 式多版本生成。借鉴 DeepSeek-R1 的 GRPO 训练范式——不是训练模型，而是在推理阶段复刻"多版本 → 组内比较 → 择优"的思路。

DSv4 的低成本使并行多版本生成在经济上可行——这是 Claude Code 做不到的（Claude API 成本太高）。

---

## 二、新增工具：grpo

```
grpo("写一个支持并发上限和超时的异步任务队列")

  → 并行 3 条路径:
     A) direct — "直接实现，一步到位"
     B) search_first — "先 grep/glob 探索现有代码，再实现"
     C) design_first — "先 ThinkBlock 充分规划，再逐步编码"

  → flash 模型做 Judge → 选出最佳 → 返回理由
```

| 特性 | 说明 |
|------|------|
| 3 路径并行 | asyncio.gather，耗时约等于最慢的一条 |
| LLM-as-Judge | flash 模型比较三条结果，选最优 + 理由 |
| 优雅降级 | Judge 不可用时回退到随机选一条成功的 |
