# Kunkun v0.7.1 更新公告

> 发布日期: 2026-07-08
> 版本: v0.7.1
> 状态: 思考控制 + AgentRuntime + AgentTeam + 评测闭环

---

## 一、版本概述

v0.7.1 的核心交付：子 Agent 线程隔离、AgentTeam 多角色协作、被动过度思考检测、评测三件套（AgentThink + AdaRubric + GRPO）。

今天的工作集中在让"多 Agent 协作"真正可用——不是简单的 spawn 子进程，而是每个子 Agent 跑在独立线程的独立 event loop 里，互不干扰。

---

## 二、Agent 基础设施

### 2.1 线程隔离 (KunkunHarness)

每个子 Agent 通过 `KunkunHarness` 跑在独立线程的独立 `asyncio` event loop 中：

```
主 Agent (主线程, loop A)
  ├── agent 工具 → KunkunHarness → Thread-1 (loop B) → AgentLoop
  ├── grpo → 3 × KunkunHarness → Thread 1-3
  ├── fastslow → 2 × KunkunHarness → Thread 1-2
  └── team → N × KunkunHarness → Thread 1-N

共享 ThreadPoolExecutor (max 8 workers)
跨线程通信: queue.Queue (线程安全)
```

解决了 v0.6 中子 Agent 和主 Agent 抢 event loop 导致的"双通道均失败"问题。

### 2.2 AgentTeam 多角色协作

```
用户任务
  → Planner 拆解 → [Explorer, Reviewer, Coder] 按依赖分批执行
  → Leader 汇总结果
```

| 角色 | 工具白名单 | 权限 |
|------|----------|------|
| Explorer | read_file, glob, grep, findsymbol, websearch | 只读 |
| Coder | + write_file, edit, bash | 读写 |
| Reviewer | + skill_load, recall | 只读+分析 |
| Planner | + grpo, agent, todowrite | 分析+设计 |

### 2.3 AgentRuntime 抽象层

```
AgentRuntime (ABC)
  ├── KunkunHarness     # 内置, DSv4
  ├── OpenCodeHarness   # 未来
  └── ExternalHarness   # 未来
```

为 FlowForge 接入外部 Harness 铺路。

## 三、思考控制（两条策略）

### 3.1 被动过度思考检测

检测三种模式，**仅记录不干预**：

- **重复操作**：同一工具 + 同一参数连续调用（读 3 个不同文件不会误判）
- **无动作循环**：连续 N 轮无 tool_call
- **犹豫信号**：thinking 中出现"maybe/不确定/让我再想想"等

每次成功工具调用后自动重置计数器。子 Agent 独立计数。

### 3.2 干预指令（检测触发时注入 SYSTEM 消息）

检测到过度思考→注入 ROLLBACK_INSTRUCTION→不影响用户界面。

> v0.7.1 移除了"思考预算强制截断"功能。实践发现 Harness 层截断流 + 注入指令的精度难以保证，负面作用大于正面。保留被动检测做日志分析。

## 五、关键修复

### 5.1 日志隔离

终端只显示 ERROR。所有 DEBUG/INFO/WARNING 写入 `.kun/logs/kunkun.log`。解决了异步日志输出干扰用户输入的问题。

### 5.2 过度思考误判修复

| 问题 | 修复 |
|------|------|
| 读 3 个不同文件触发"重复操作" | 改为同一工具 + 同一参数才算完全重复 |
| hesitation_count 永不重置 | 每次成功工具调用后清零 |
| 父 Agent 编排子 Agent 后被误判重复 | agent/grpo/fastslow/team 工具返回后重置检测器 |

### 5.3 线程池竞争

`asyncio.to_thread` 使用默认线程池 → 改为自定义 `ThreadPoolExecutor` 的 `run_in_executor`。线程池从 4 扩到 8。
