## 一、版本概述

v0.3 引入了 Skill 系统——开发者可以在 `skills/` 目录下预置领域知识模板，Kun 启动时自动扫描并按用户 prompt 触发词匹配注入 System Prompt。

Skill 和 v0.2 的 Memory 互为补充：
- **Memory**: 项目运行时数据，由 Agent 通过 `remember` 工具写入 `.kun/memory/`
- **Skill**: 预置领域知识，由开发者写入 `skills/` 目录，启动时加载

两者使用相同的 `.md` + YAML frontmatter 格式，通过 System Prompt 注入。

## 三、Skill 系统架构

### 文件格式

```yaml
---
name: code-review
description: 中文代码审查 Skill
triggers:
  - 代码审查
  - code review
  - 检查代码
---

## 审查维度
...
```

### 匹配策略

1. 启动时扫描 `skills/` 目录，递归加载所有 `SKILL.md`
2. 每次对话，遍历用户 prompt 匹配所有 Skill 的 `triggers` 关键词
3. 按命中数排序，取前 3 条注入 System Prompt
4. 匹配大小写不敏感

### 预置 Skill（3 个）

| Skill | 触发词示例 | 内容 |
|-------|----------|------|
| `code-review` | 代码审查、review、检查代码 | 5 维度审查 (命名/逻辑/安全/维护/性能) + 表格输出格式 |
| `python-project` | Python、写代码、编码规范 | PEP 8 风格、类型注解、docstring、项目结构、错误处理 |
| `git-conventions` | git、commit、提交 | Conventional Commits + 中文提交信息 + 分支策略 |

## 四、遇到的问题与解决方案

### 问题 1: Skill 加载数量为 0

**现象**: 使用 `SkillLoader(skill_dir="skills")` 加载预置 Skill，返回空列表。

**解决**: 检查发现 `skills/` 目录在项目根目录，但 Agent 运行时 `cwd` 可能在别处。修改加载逻辑：目录不存在时返回空列表而非报错，并在 AgentLoop 中传入 `config.skill_dir`（从配置读取，默认 `"skills"`）。

### 问题 2: Skill 注入时机

**现象**: Skill 在 `__init__` 中加载，但 `run()` 中才匹配 prompt，导致同一会话多次 `run()` 时 Skill 列表可能过期。

**解决**: 每次 `run()` 开始时调用 `self.skills.load()` 重新加载（如果用户手动新增了 Skill 文件），然后按当前 prompt 匹配。

## 五、自定义 Skill

在 `skills/` 下创建新目录和 `SKILL.md`：

```bash
mkdir -p skills/my-skill
cat > skills/my-skill/SKILL.md << 'EOF'
---
name: my-skill
description: 我的自定义 Skill
triggers:
  - 关键词1
  - keyword2
---

## 规则

1. 第一条规则
2. 第二条规则
EOF
```

下次启动 Kun 时自动生效，无需修改代码。
